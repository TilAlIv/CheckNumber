В проекте OCP реализовал "принцип открытости/закрытости".
В данном принципе смысл заключается в том, что есть некий основной интерфейс в котором задано шаблонное действие без реализации.
Каждый наследуемый класс от данного интерфеса, обязан сделать свю реализацию этого шаблонного действия, это т.н. полиморфизм.
На один интерфейс можно делать разные реализации. 

Есть интерфейс IGame c методом start, есть наследник класс Game, где происходит реализация шаблонного метода start от
интерфйеса IGame. Есть также класс StartGame, в который можно передовать всех наследуемых наследников от интерфейса
IGame. В классе Programm происходит запуск экземляра класса.
