В проекте DIP реализовал "принцип инверсий зависимости".

Данный принцип также достаточно прост как и ISP, весь необходимый функционал разделяем по интерфейсам. После чего формируем нужный
класс необходимым количеством интерфейсов, т.е. на выбранный класс навешиваем нужный функционал.

Создал два интерфейса ICheckbale и IStartable, создал класс Game в котором есть реализация этих интерфейсов.
